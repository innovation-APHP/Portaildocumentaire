🚀 COMMANDE DE LANCEMENT
═══════════════════════════════════════════════════════════════════════════════

Tu es maintenant prêt à lancer Docker ! Voici les étapes :

1️⃣ Vérifie que Dockerfile et VERSION sont bien des FICHIERS (pas des répertoires)
   
   ls -lh Dockerfile VERSION
   
   Si ce sont des répertoires (drwx...), supprime-les et recrée-les :
   
   rm -rf Dockerfile VERSION
   
   # Recrée Dockerfile avec le contenu que je t'ai fourni plus haut
   # Recrée VERSION avec : echo "1.0.0" > VERSION

2️⃣ Édite le fichier .env si nécessaire
   
   nano .env
   
   Change au minimum :
   - POSTGRES_PASSWORD=ton_mot_de_passe_securise

3️⃣ Lance Docker Compose
   
   # Stack complet (Portail + Wiki.js + PostgreSQL)
   docker-compose up -d --build
   
   # OU uniquement le Portail (si Wiki.js existe ailleurs)
   docker-compose up -d --build portail-docs

4️⃣ Vérifie le déploiement
   
   docker-compose ps
   docker-compose logs -f portail-docs

5️⃣ Accède à l'application
   
   Portail : http://localhost:8080
   Wiki.js : http://localhost:3000

💡 PROBLÈMES CONNUS ET SOLUTIONS
═══════════════════════════════════════════════════════════════════════════════

❌ Dockerfile est un répertoire au lieu d'un fichier
   
   Solution :
   rm -rf Dockerfile VERSION
   # Puis recrée-les comme des fichiers

❌ .env.docker: No such file or directory
   
   Solution :
   cp .env.docker .env
   # Ou utilise directement .env qui existe déjà

❌ Port already in use
   
   Solution :
   # Change les ports dans .env
   PORTAIL_PORT=8081
   WIKIJS_PORT=3001

❌ Permission denied sur docker-entrypoint.sh
   
   Solution :
   chmod +x docker/docker-entrypoint.sh
